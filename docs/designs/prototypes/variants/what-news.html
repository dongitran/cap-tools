Current capability map

What News

SAP Tools brings the core SAP BTP developer workflow into one VS Code surface: pick scope, stream logs, map apps, export artifacts, and run HANA SQL from the selected app context.

01

Scope

Select and restore BTP scope

Login once, pick area, region, org, and space, or jump through Quick Org Search when synced topology is available.

  • Region catalog with extension landscapes
  • Cached org, space, and app loading
  • Confirmed scope restore after reload
  • Sync interval, manual sync, and logout settings
02

Logs

Stream app logs in CFLogs

Start logging from the sidebar and inspect app output in a dedicated bottom panel beside Output and Terminal.

  • App, level, and message filters
  • Recent fetch and live stream states
  • Column, font size, and log limit controls
  • Click a row to copy its message
03

Apps

Map services to local folders

Connect running CF services to local workspace folders so generated files land next to the matching service code.

  • Root folder scan and conflict selection
  • default-env.json and pnpm-lock.yaml export
  • SQLTools HANA connection export
  • Confirmation before writing sensitive files
04

SQL

Run app-scoped HANA SQL

Open app-specific SQL files, discover HANA tables, run selected SQL, and review result tabs without leaving VS Code.

  • Table discovery and readable table names
  • Completions for SQL keywords and tables
  • Quick SELECT plus manual run shortcuts
  • CSV, JSON, row, and cell copy actions
